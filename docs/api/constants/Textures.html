Texture （纹理）常量

Mapping Modes（映射模式）

THREE.UVMapping THREE.CubeReflectionMapping THREE.CubeRefractionMapping THREE.EquirectangularReflectionMapping THREE.EquirectangularRefractionMapping THREE.SphericalReflectionMapping THREE.CubeUVReflectionMapping THREE.CubeUVRefractionMapping
定义了纹理的映射模式。
[page:Constant UVMapping] 为默认值，使用 mesh（网格）的 UV coordinates（UV坐标）映射纹理。

其余的定义环境映射类型。

[page:Constant CubeReflectionMapping] 和 [page:Constant CubeRefractionMapping] 适用于 [page:CubeTexture CubeTexture]， 由六个纹理组成，每个纹理对应一个面。
[page:Constant CubeReflectionMapping] 是 [page:CubeTexture CubeTexture] 的默认值。

[page:Constant EquirectangularReflectionMapping] 和 [page:Constant EquirectangularRefractionMapping] 适用于 equirectangular 环境贴图。 也称为 lat-long map（经纬映射）， equirectangular 纹理表示沿水平中心线的 360 度视图，以及沿垂直轴的180度视图， 图像的顶部和底部边缘分别映射球体的北极和南极。

[page:Constant SphericalReflectionMapping] 专用于 spherical reflection map（球形反射贴图）， 可通过裁剪 mirrored ball（镜面球）的图片得到。无论相机与立方体贴图对象或表面的位置如何相对， 球体贴图都将呈现为“面向”相机。

查看示例 [example:webgl_materials_envmaps materials / envmaps]。

Wrapping Modes（包装模式）

THREE.RepeatWrapping THREE.ClampToEdgeWrapping THREE.MirroredRepeatWrapping
定义了纹理的 [page:Texture.wrapS wrapS] 和 [page:Texture.wrapT wrapT] 属性， 它们分别定义了水平和垂直纹理包装。

使用 [page:constant RepeatWrapping]，纹理将简单的无限重复。

[page:constant ClampToEdgeWrapping] 为默认值。 纹理最后的像素将延伸至网格的边缘。

使用 [page:constant MirroredRepeatWrapping]，纹理将无限镜像重复。

Magnification Filters（放大滤镜）

THREE.NearestFilter THREE.LinearFilter
与纹理的 [page:Texture.magFilter magFilter] 属性一同使用， 这些属性定义了当纹理像素映射到小于或等于一个纹理元素（texel）的区域时要使用的放大函数。

[page:constant NearestFilter] 返回与指定纹理坐标最接近（在曼哈顿距离内）的纹理元素的值。

[page:constant LinearFilter] 为默认值， 它返回最接近指定纹理坐标的四个纹理元素的加权平均值， 并且包含被纹理的其他部分包装或重复的子项， 具体取决于 [page:Texture.wrapS wrapS] 和 [page:Texture.wrapT wrapT] ，以及确切的映射值。

Minification Filters（缩小过滤器）

THREE.NearestFilter THREE.NearestMipMapNearestFilter THREE.NearestMipMapLinearFilter THREE.LinearFilter THREE.LinearMipMapNearestFilter THREE.LinearMipMapLinearFilter
与纹理的 [page:Texture.minFilter minFilter] 属性一同使用， 这些属性定义了当纹理像素映射到大于一个纹理元素（texel）的区域时要使用的缩小函数。

除了 [page:constant NearestFilter] 和 [page:constant LinearFilter]， 还可以使用以下的四个函数进行缩小：

[page:constant NearestMipMapNearestFilter] 选择最接近映射纹理像素大小的 mipmap， 使用 [page:constant NearestFilter] 标准（最接近像素中心的纹理元素）来生成纹理值。

[page:constant NearestMipMapLinearFilter] 选择两个与纹理像素大小最匹配的 mipmap 使用 [page:constant NearestFilter] 标准从每个 mipmap 生成纹理值。 最终的纹理值是这两个值的加权平均值。

[page:constant LinearMipMapNearestFilter] 选择最接近映射纹理像素大小的 mipmap， 使用 [page:constant LinearFilter] 标准 （最接近像素中心的四个纹素的加权平均值）来生成纹理值。

[page:constant LinearMipMapLinearFilter] 为默认值，选择两个与纹理像素大小最匹配的 mipmap 使用 [page:constant LinearFilter] 标准从每个 mipmap 生成纹理值。 最终的纹理值是这两个值的加权平均值。

参考示例 [example:webgl_materials_texture_filters materials / texture / filters]。

类型

THREE.UnsignedByteType THREE.ByteType THREE.ShortType THREE.UnsignedShortType THREE.IntType THREE.UnsignedIntType THREE.FloatType THREE.HalfFloatType THREE.UnsignedShort4444Type THREE.UnsignedShort5551Type THREE.UnsignedShort565Type THREE.UnsignedInt248Type
适用于纹理的 [page:Texture.type type] 属性，该属性必须与正确的格式相对应。参阅下文了解详情。

[page:constant UnsignedByteType] 为默认值。

格式

THREE.AlphaFormat THREE.RGBFormat THREE.RGBAFormat THREE.LuminanceFormat THREE.LuminanceAlphaFormat THREE.RGBEFormat THREE.DepthFormat THREE.DepthStencilFormat
与纹理的 [page:Texture.format format] 属性一同使用， 这些属性定义 shaders （着色器）如何读取 2D 纹理（即 *texels* ）。

[page:constant AlphaFormat] 忽略红，绿和蓝值，只读取 alpha 值。 [page:Texture.type type] 必须设置为 [page:constant UnsignedByteType]。

[page:constant RGBFormat] 忽略 alpha 值，只读取红，绿和蓝值。 [page:Texture.type type] 必须设置为 [page:constant UnsignedByteType] 或 [page:constant UnsignedShort565Type]。

[page:constant RGBAFormat] 为默认值，读取红，绿，蓝和 alpha 值。 [page:Texture.type type] 必须设置为 [page:constant UnsignedByteType], [page:constant UnsignedShort4444Type] 或 [page:constant THREE.UnsignedShort5551Type]。

[page:constant LuminanceFormat] 将每个元素读取为独立 luminance（亮度值）。 然后将其转换为范围在 [0,1] 的浮点数， 然后通过将亮度加到红，绿和蓝通道中，并将Alpha通道设置为 1.0，将其组装成 RGBA 元素。 [page:Texture.type type] 必须设置为 [page:constant UnsignedByteType]。

[page:constant LuminanceAlphaFormat] 将每个元素读取为亮度/alpha 值。 除了 alpha 通道可能设置 *1.0* 以外的值，处理过程与 [page:constant LuminanceFormat] 相同。 [page:Texture.type type] 必须设置为 [page:constant UnsignedByteType]。

[page:constant RGBEFormat] 与 [page:constant RGBAFormat] 相同。

[page:constant DepthFormat] 将每个元素读取为一个 depth （深度值），并将其转换为范围在 [0,1] 的浮点数， [page:Texture.type type] 必须设置为 [page:constant UnsignedIntType] 或 [page:constant UnsignedShortType]。 这是 [page:DepthTexture DepthTexture] 的默认值。

[page:constant DepthStencilFormat] 将每个元素作为一对 depth （深度）和 stencil（模板）值。 其中深度值将按 [page:constant DepthFormat] 读取。 模板值基于深度 + 模板的内部格式进行读取。 [page:Texture.type type] 必须设置为 [page:constant UnsignedInt248Type]。

注意，纹理必须有如上的所述的正确 [page:Texture.type type] 值， 详细信息，参考 此页面。

DDS / ST3C 压缩纹理格式

THREE.RGB_S3TC_DXT1_Format THREE.RGBA_S3TC_DXT1_Format THREE.RGBA_S3TC_DXT3_Format THREE.RGBA_S3TC_DXT5_Format
与 [page:CompressedTexture CompressedTexture] 的 [page:Texture.format format] 属性一同使用， 这些属性需要 WEBGL_compressed_texture_s3tc 扩展的支持。
据 WebglStats 统计，至 2016 年 2 月，超过 80% WebGL 设备支持此扩展。

该扩展有四种 S3TC 格式可用。分别是：
[page:constant RGB_S3TC_DXT1_Format]：RGB 图像格式的 DXT1 压缩图像
[page:constant RGBA_S3TC_DXT1_Format]：RGB 图像格式的 DXT1 压缩图像，有简单的 alpha 值开/关。
[page:constant RGBA_S3TC_DXT3_Format]：RGBA 图像格式的 DXT3 压缩图像。与 32 位 RGBA 纹理相比，提供 4:1 压缩。
[page:constant RGBA_S3TC_DXT5_Format]：RGBA 图像格式的 DXT5 压缩图像。它还提供 4:1 压缩，但与 DXT3 压缩的不同之处在于如何对 alpha 值压缩。

PVRTC 压缩纹理格式

THREE.RGB_PVRTC_4BPPV1_Format THREE.RGB_PVRTC_2BPPV1_Format THREE.RGBA_PVRTC_4BPPV1_Format THREE.RGBA_PVRTC_2BPPV1_Format
与 [page:CompressedTexture CompressedTexture] 的 [page:Texture.format format] 属性一同使用， 这些属性需要 WEBGL_compressed_texture_pvrtc 扩展的支持。
据 WebglStats 统计， 至 2016 年 2 月，不足 8% WebGL 设备支持此扩展。 PVRTC 仅适用于具有 PowerVR 芯片组的移动设备， 主要是 Apple 设备。

该扩展有四种 PVRTC 格式可用。分别是：
[page:constant RGB_PVRTC_4BPPV1_Format]：4位模式下的RGB压缩。每 4×4 像素一块。
[page:constant RGB_PVRTC_2BPPV1_Format]：2位模式下的RGB压缩。每 8×4 像素一块。
[page:constant RGBA_PVRTC_4BPPV1_Format]：4位模式下的RGBA压缩。每 4×4 像素一块。
[page:constant RGBA_PVRTC_2BPPV1_Format]：2位模式下的RGBA压缩。每 8×4 像素一块。

ETC 压缩纹理格式

THREE.RGB_ETC1_Format
与 [page:CompressedTexture CompressedTexture] 的 [page:Texture.format format] 属性一同使用， 这些属性需要 WEBGL_compressed_texture_etc1 扩展的支持。
据 WebglStats 统计，至 2016 年 2 月， 仅有 13% 左右的 WebGL 设备支持此扩展。

编码

THREE.LinearEncoding THREE.sRGBEncoding THREE.GammaEncoding THREE.RGBEEncoding THREE.LogLuvEncoding THREE.RGBM7Encoding THREE.RGBM16Encoding THREE.RGBDEncoding THREE.BasicDepthPacking THREE.RGBADepthPacking
与 [page:Texture.encoding encoding] 属性一同使用.

当在材质已应用纹理后再更改编码类型， 则需要设置 [page:Material.needsUpdate Material.needsUpdate] 为 *true* 使材质重新编译。

[page:constant LinearEncoding] 为默认值。 除此之外的其他值仅适用于材质的贴图，envMap 和 emissiveMap。
