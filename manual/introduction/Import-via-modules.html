虽然通过 script 标签导入 three.js 是快速起步和运行的好方法，但对于长期项目有一些缺点，例如：
  • 你必须手动获取并引入库的副本作为项目源代码的一部分
  • 更新库的版本需要手动操作
  • 在检入新版本的库时，你的版本控制对比会被构建文件的许多行混淆
使用像 npm 这样的依赖管理工具可以避免这些限制，你只需下载所需版本的库并将其导入到你的机器上即可。

通过 npm 安装

Three.js 已作为一个 npm 模块发布, 请参阅: npm。这意味着你只需运行“npm install three”就能将 three.js 导入到你的项目中

导入模块

如果你使用 Webpack 或者 Browserify 等工具打包你的文件，你可以“打包所有依赖项并在浏览器中使用 require('modules') 。”
你现在应该能够将模块导入到你的源文件中，并继续照常使用它。
var THREE = require('three'); var scene = new THREE.Scene(); ...
你也可以利用 ES6 import 语法:
import * as THREE from 'three'; const scene = new THREE.Scene(); ...
或者你只想导入 three.js 库的选定部分，例如 Scene：
import { Scene } from 'three'; const scene = new Scene(); ...

注意事项

目前无法通过这种方式导入“examples/js”目录中的文件。 因为部分文件依赖于全局命名空间污染 THREE。更多信息请参阅 Transform `examples/js` to support modules #9562.
